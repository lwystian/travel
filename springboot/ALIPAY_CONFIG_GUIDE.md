# 支付宝电脑网站支付配置指南

## 一、支付宝开放平台配置

### 1. 注册开发者账号
1. 访问 [支付宝开放平台](https://open.alipay.com/)
2. 注册并登录开发者账号

### 2. 创建应用
1. 进入「控制台」→「我的应用」
2. 点击「创建应用」
3. 选择「自研服务」→「网页/移动应用」
4. 填写应用信息

### 3. 签约电脑网站支付
1. 在应用详情中找到「产品绑定」
2. 绑定「电脑网站支付」产品
3. 完成签约（需要企业资质或选择沙箱测试）

### 4. 获取配置信息
在「开发设置」→「接口加签方式」中获取：

| 配置项 | 说明 |
|--------|------|
| APPID | 应用ID，在应用概览页面获取 |
| 商户私钥 | 使用支付宝密钥工具生成，PKCS8格式 |
| 支付宝公钥 | 生成的公钥填入开放平台 |

## 二、后端配置

### 1. 修改 application.properties

找到 `src/main/resources/application.properties`，修改以下配置：

```properties
# 是否沙箱环境（测试时设为true，上线时设为false）
alipay.sandbox=true

# 你的应用ID
alipay.appId=2021001234567890

# 商户私钥（PKCS8格式，注意保留换行）
alipay.privateKey=
-----BEGIN PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC...
（你的私钥内容）
-----END PRIVATE KEY-----

# 支付宝公钥
alipay.alipayPublicKey=
-----BEGIN PUBLIC KEY-----
MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...
（支付宝公钥内容）
-----END PUBLIC KEY-----

# 同步回调地址（支付成功后跳转的页面）
alipay.returnUrl=http://你的域名/payment/result

# 异步通知地址（需要公网可访问）
alipay.notifyUrl=http://你的域名/api/tour-order-pay/notify
```

### 2. 配置说明

| 配置项 | 说明 | 示例 |
|--------|------|------|
| sandbox | 是否使用沙箱环境 | `true`=沙箱测试，`false`=正式环境 |
| appId | 支付宝应用ID | `2021001234567890` |
| privateKey | 商户私钥 | PKCS8格式，保留格式和换行 |
| alipayPublicKey | 支付宝公钥 | `-----BEGIN PUBLIC KEY-----` 开头 |
| returnUrl | 同步回调地址 | 用户支付后跳转的页面 |
| notifyUrl | 异步通知地址 | 后端接收支付结果的接口 |

### 3. 公网访问配置

异步通知接口（`/api/tour-order-pay/notify`）需要公网可访问：

**方式一：内网穿透**
```bash
# 使用 natapp 或 ngrok
ngrok http 1236
```

**方式二：部署到云服务器**
```bash
# 将应用部署到有公网IP的服务器
java -jar springboot.jar --server.port=1236
```

**方式三：配置nginx反向代理**
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /api/ {
        proxy_pass http://localhost:1236/;
    }
}
```

## 三、沙箱测试

### 1. 启用沙箱环境
```properties
alipay.sandbox=true
```

### 2. 获取沙箱配置
1. 登录 [沙箱环境](https://openhome.alipay.com/develop/sandbox/app)
2. 获取沙箱 APPID 和密钥
3. 下载「支付宝沙箱」App 进行扫码支付测试

### 3. 沙箱与正式环境的区别

| 项目 | 沙箱环境 | 正式环境 |
|------|----------|----------|
| 网关地址 | openapi-sandbox.dl.alipaydev.com | openapi.alipay.com |
| 支付 | 使用沙箱版支付宝App | 使用真实支付宝App |
| 资金 | 不真实扣款 | 真实扣款 |
| 通知 | 可能有延迟 | 正常通知 |

## 五、常见问题

### Q1: 签名验证失败
**可能原因：**
1. 私钥格式不正确（需要PKCS8）
2. 公钥没有正确配置到开放平台
3. 编码格式不匹配

**解决方案：**
```java
// 确保使用正确的编码
alipay.charset=UTF-8
```

### Q2: 异步通知接收不到
**可能原因：**
1. 通知地址不可公网访问
2. 防火墙阻断了请求
3. 接口响应格式错误

**解决方案：**
1. 使用 `curl` 测试公网访问
2. 检查服务器防火墙
3. 确保接口返回 `success` 或 `fail`

### Q3: 订单重复更新
**已实现的防护：**
1. 使用 notify_id 去重（Redis）
2. 检查订单状态后再更新
3. 金额校验

### Q4: 支付金额与订单金额不符
**可能原因：**
1. 金额单位不一致（元 vs 分）
2. 货币类型问题

**已实现的防护：**
1. 后端金额校验（允许0.01元误差）
2. 金额格式统一为字符串

## 七、API接口说明

### 后端接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/tour-order-pay/pay/{orderId}` | GET | 生成支付表单 |
| `/api/tour-order-pay/status/{orderId}` | GET | 获取订单支付状态 |
| `/api/tour-order-pay/order` | GET | 根据订单号获取订单 |
| `/api/tour-order-pay/notify` | POST | 支付宝异步通知 |
| `/api/tour-order-pay/return` | GET | 支付宝同步回调 |
| `/api/tour-order-pay/mock-pay/{orderId}` | POST | 模拟支付（测试用） |

### 前端页面

| 页面路径 | 说明 |
|----------|------|
| `/tour-order-pay/:id` | 行程订单支付页面 |
| `/payment/result` | 支付结果页面 |
